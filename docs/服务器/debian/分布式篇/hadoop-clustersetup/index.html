    Hadoop 3.4.1 集群版之HDFS #

    https://blog.csdn.net/tang5615/article/details/120382513

    root操作(建议docker打包) # 

    sed -i "s|bookworm-updates|bookworm-updates bullseye|g" /etc/apt/sources.list.d/debian.sources
apt update && apt install -y openjdk-11-jdk
sudo apt install openjdk-11-jdk-headless -y
sed -i '$aPATH=\$PATH:/usr/games:/opt/hadoop-3.4.1/bin:/opt/hadoop-3.4.1/sbin' /etc/profile
sed -i '$aexport JAVA_HOME=/usr/lib/jvm/java-11-openjdk-amd64/' /etc/profile
sudo useradd -ms/bin/bash -k/etc/skel/ hadoop
sudo usermod -aG sudo hadoop
echo hadoop:. | chpasswd
cd /opt
wget https://dlcdn.apache.org/hadoop/common/hadoop-3.4.1/hadoop-3.4.1.tar.gz
tar -zxvf hadoop-3.4.1.tar.gz
sed -i "1 i JAVA_HOME=/usr/lib/jvm/java-11-openjdk-amd64" /opt/hadoop-3.4.1/etc/hadoop/hadoop-env.sh
mv /opt/hadoop-3.4.1.tar.gz /opt/hadoop-3.4.1
find /opt/hadoop-3.4.1/ -name *cmd -delete
mkdir /opt/hadoop-3.4.1/data
chown hadoop /opt -R

    免登录 #

    • 互相之间都要免登录,否则无法全部启动,之前可以不登录

    修改配置 #

    • /opt/hadoop-3.4.1/etc/hadoop/core-site.xml
    sed -i "20 i <property><name>fs.defaultFS</name><value>hdfs://200.mm:9000</value></property>" /opt/hadoop-3.4.1/etc/hadoop/core-site.xml
sed -i "20 i <property><name>hadoop.http.staticuser.user</name><value>hadoop</value></property>" /opt/hadoop-3.4.1/etc/hadoop/core-site.xml
sed -i "20 i <property><name>hadoop.tmp.dir</name><value>/opt/hadoop-3.4.1/data/</value></property>" /opt/hadoop-3.4.1/etc/hadoop/core-site.xml
    • /opt/hadoop-3.4.1/etc/hadoop/hdfs-site.xml
    sed -i "20 i <property><name>dfs.replication</name><value>2</value></property>" /opt/hadoop-3.4.1/etc/hadoop/hdfs-site.xml
sed -i "20 i <property><name>dfs.datanode.data.dir</name><value>/opt/hadoop-3.4.1/data/</value></property>" /opt/hadoop-3.4.1/etc/hadoop/hdfs-site.xml
    • /opt/hadoop-3.4.1/etc/hadoop/workers
    echo -e '200.mm\n201.mm\n202.mm' > /opt/hadoop-3.4.1/etc/hadoop/workers

    主机上启动 #

    • 先格式化
    hdfs namenode -format
start-dfs.sh

    添加节点 #

    • 新节点重复以上操作
    • master登录一次新节点,确保免登录生效
    • masterd的workers记录要更新
    • 在新的机器执行
    hdfs --daemon start datanode
echo -e '200.mm\n201.mm\n202.mm\n203.mm' > /opt/hadoop-3.4.1/etc/hadoop/workers
